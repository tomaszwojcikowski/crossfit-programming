# Functional Fitness Lesson Planning Reference

Last updated: 2 April 2026

Purpose: distill the attached long-form analysis into practical rules for future
CrossFit class design in this repo.

## What this document is for

- Use this as a planning reference when writing future daily workouts.
- Keep the useful high-level coaching and physiology ideas.
- Filter them through this repo's existing standards in [agents.md](../agents.md).
- Prefer practical class design over academic completeness.

## Core takeaway

The source analysis is directionally right on the big ideas:

- A good class is not random exercise selection.
- The hour needs a clear structure.
- Warm-up, teaching, loading, conditioning, and cooldown should each have a job.
- Scaling should preserve stimulus, not just make the workout easier.
- Logistics matter as much as physiology in a group class.

For this repo, the main value is not the exact affiliate templates discussed in the
source. The value is the planning logic behind them.

## Repo-standard class architecture

When using ideas from the source, default to the repo's operating model:

- Hard 60-minute cap.
- Single group start.
- One lane staged before class starts.
- Warm-up 8-12 min.
- Post-warmup mobility 60-90s.
- Strength or skill 10-20 min when needed.
- Reset window 3-5 min when needed.
- Movement prep 2-4 min after strength/skill and before conditioning.
- Main conditioning 12-25 min.
- Cooldown 2-4 min.

This is the most important override to the source material. Some external models
allow longer timelines, staggered starts, or more open-ended transitions. This repo
does not.

## Whiteboard brief principles

The source correctly emphasizes that the brief sets safety, clarity, and pacing.
For future workout writing, keep the brief short and answer four questions only:

1. What are the movements?
2. What should the workout feel like?
3. What are the scaling options?
4. How will the room flow?

Practical rules:

- Keep whiteboard time at 3 minutes or less.
- Define stimulus in one sentence.
- State top safety risk and simplest scale.
- Explain room flow before athletes touch equipment.
- Avoid long theory lectures.

Useful source insight:

- Athletes retain strategy better when they know whether the piece is a sprint,
  grind, interval, or sustained aerobic effort.

## Warm-up and movement prep

The source is strongest when it separates general readiness from specific skill
rehearsal. That fits this repo well.

### What to keep

- Use RAMP structure.
- Progress intensity across rounds.
- Prepare the exact joints and patterns needed later.
- Use technical primers to teach positions before load or fatigue rises.
- Use dynamic prep before training, not long static stretching.

### Repo application

- Write warm-ups as R1 / R2 / R3.
- Keep cardio as a clear time block with rising intensity.
- Prefer bodyweight or a single extra implement.
- Keep barbell work out of the main warm-up when possible.
- Put empty-bar or light-implement rehearsal in Movement Prep instead.

### On specialized barbell warm-ups

The source highlights Burgener-style progressions. Use the underlying lesson, not
the full ritual every time:

- Break complex lifts into a few teachable positions.
- Move from simple to fast.
- Use low-rep, high-quality technical rehearsal.
- Stop before fatigue reduces precision.

Use this for snatch, clean, jerk, and gymnastics progressions when the day actually
demands technical precision.

## Strength, skill, and conditioning interplay

The source reinforces a useful programming principle: the class has to make room
for the primary adaptation of the day.

Future programming rules:

- If strength or skill is the priority, shorten the metcon.
- If long aerobic work is the priority, reduce or remove the strength block.
- If heavy loading is used to potentiate later power output, leave 3-7 minutes
  before the high-intent effort.
- Use the reset window to regroup the class and stage equipment.
- Use movement prep after the reset so athletes hit the conditioning piece with
  fresh, relevant reps.

This aligns well with the repo's existing PAPE guidance.

## Scaling philosophy

The source is correct that scaling is about preserving intent.

Keep these rules:

- Scale load first.
- Scale volume second.
- Scale movement complexity third.
- Use range-of-motion limits only when needed for pain or restriction.
- Prefer a harder but simpler variation over an easier technical variation.

Practical examples for future workouts:

- Heavy front squat scales to lighter front squat before air squat.
- Strict pulling scales to banded strict pulling before kipping.
- Heavy deadlift sprint scales to moderate deadlift or similar hinge, not a slow,
  unrelated substitute.
- Handstand push-up volume can scale to heavy dumbbell press if the intent is fast,
  forceful upper-body pressing.

## Injury and pain management

One of the most useful parts of the source is the reminder that not every injured
athlete needs total rest. For class planning, that means writing obvious swaps.

Useful patterns:

- Change leverage before removing the pattern.
- Change implement before removing load.
- Reduce ROM before removing the movement entirely.
- Slow tempo to create training effect with lighter loads.
- Use cyclical conditioning to preserve engine when local loading is limited.

Examples worth keeping in mind:

- Back squat to front squat when torso position matters.
- Barbell press to dumbbell or landmine press for freer shoulder motion.
- Bench or dip to floor press when end-range shoulder extension is the issue.
- Tempo squats, presses, or hinges to maintain strength stimulus with lower load.

Programming implication:

- Future daily docs should continue to include simple A/B scaling that protects the
  intended stimulus for common shoulder, knee, hip, and front rack restrictions.

## Population-specific adjustments

The source covers different populations well. For this repo, the most useful parts
are Masters and beginners.

### Beginners

- Keep movement choices simple.
- Limit technical fatigue.
- Give clear, prescriptive scales.
- Use submaximal loading and repeatable positions.

### Masters

- Extend warm-up and build-up slightly within the same 60-minute cap.
- Reduce unnecessary volume.
- Favor the minimum effective dose.
- Use time caps and AMRAP structures to control total exposure.
- Prioritize tissue tolerance, balance, and consistency over peak output.

### Youth

The youth section is useful conceptually, but this repo is built around standard
group CrossFit classes. Borrow only these broad ideas if needed:

- Short explanations.
- More play, less lecture.
- Emphasis on movement literacy and coordination.

## Logistics and class flow

The source is right that class design fails when the floor plan fails.

Keep these planning checks:

- Stage all equipment before class.
- Protect central space for the highest-risk or most dynamic movement.
- Avoid machine layouts that split coaching attention.
- Prevent athletes from crossing traffic lines during fatigue.
- Fix bottlenecks in the written plan, not mid-workout if possible.

Important repo override:

- Do not design around heats or staggered starts as the default solution.
- Solve crowding with smarter equipment choices, lower-complexity substitutions,
  rep adjustments, shared stations, or lane planning.

## Affiliate programming takeaways

The source compares CAP, Mayhem, NCFIT, and CompTrain. The exact brand system is
less important than the transferable lessons.

Useful takeaways:

- CAP: coach education and clean class timelines matter.
- Mayhem: strong scaling ladders and clear strength progressions help classes run.
- NCFIT: pacing targets and strict time discipline are valuable.
- CompTrain: programming should work for the whole room, not just the top 10%.

Best use in this repo:

- Borrow structure, not branding.
- Write workouts for a mixed-ability class first.
- Keep competitor-style details only when they improve the general class outcome.

## What to borrow from the source

- Class planning is a sequencing problem, not just exercise selection.
- The warm-up should prepare both physiology and skill.
- The brief should define stimulus and logistics clearly.
- Scaling should preserve time domain and movement pattern.
- Injury modifications should keep athletes training when possible.
- Masters athletes usually need less volume and more preparation.
- Equipment layout is part of programming.

## What not to import blindly

- Long whiteboard explanations.
- Overly dense class plans that are hard to coach live.
- Multi-hour logic borrowed from competitor programs.
- Default use of staggered starts or pod systems.
- Excessive specialized drills when a simpler primer will do.
- Bodybuilding-style accessory add-ons inside a standard 60-minute class unless the
  day's structure clearly supports them.

## Workout-writing checklist

Before finalizing a daily workout, check:

1. Is the primary adaptation of the day obvious?
2. Does the timeline fit a real 60-minute class?
3. Can the whole room start together?
4. Does the warm-up prepare the actual demands of the day?
5. Is there a clear movement-prep bridge into the conditioning piece?
6. Are A/B scales enough to preserve stimulus for most athletes?
7. Are equipment bottlenecks solved on paper?
8. Is the expected finish window or rounds estimate included?
9. Are the cues external, short, and coachable under fatigue?
10. Will this still read clearly on a phone?

## Bottom line for this repo

Use the source analysis as confirmation of the existing direction in this repo, not
as a reason to make daily programming more complicated.

The most durable lesson is simple:

- clear timeline
- clear stimulus
- clear scaling
- clear logistics
- enough prep to move well
- enough structure to stay inside 60 minutes

If a future workout follows those rules, it will usually be class-ready.
